# Zapisi.com (Клон Cal.com) — Сервис онлайн-записи
Данный сервис является учебным и является практической работой с ИИ-агентом. Весь код и описания были выполнены агентом.

Современный и отзывчивый веб-интерфейс для системы онлайн-записи на приёмы к врачам, барберам, мастерам маникюра или любым другим специалистам. Проект разработан на базе **React**, **Vite**, **TypeScript** и **Mantine UI**.

Интерфейс спроектирован по микросервисной архитектуре и работает по строгому API-контракту, описанному с использованием **TypeSpec** (`main.tsp`) и скомпилированному в спецификацию **OpenAPI 3.0** (`openapi.yaml`).

---

## Архитектура и стек технологий

* **Сборщик и Dev-сервер:** [Vite](https://vite.dev/) (высокая скорость HMR и сборки).
* **Библиотека интерфейса:** [Mantine UI v7](https://mantine.dev/) (продвинутые компоненты календаря, форм, уведомлений).
* **Стилизация:** CSS Modules + PostCSS.
* **Управление состоянием и кэширование:** [TanStack Query v5 (React Query)](https://tanstack.com/query/latest) — для эффективного и отказоустойчивого взаимодействия с сервером.
* **API-клиент:** Axios с полной типизацией моделей запросов и ответов.
* **Локальный эмулятор (Mock):** [Stoplight Prism](https://stoplight.io/open-source/prism) — для разработки без запущенного реального бэкенда.

## Возможности приложения

### 👤 Публичная часть (для Гостя):
1. **Главная страница (`/`):** Просмотр списка доступных услуг и специалистов (название, длительность встречи, часы работы).
2. **Календарь записи (`/book/:id`):** Выбор даты в пределах ближайших **14 дней** (с ограничением прошлых и слишком далёких дат).
3. **Сетка слотов:** Интерактивное отображение сгенерированных свободных и занятых (`reserved: true`) слотов времени для выбранного дня.
4. **Форма бронирования:** Удобное модальное окно для ввода Имени, Фамилии и Номера телефона.
5. **Экран подтверждения:** Красивая карточка с кодом бронирования, выбранной датой, временем и контактами.

### 👑 Панель управления (для Мастера/Врача):
1. **Авторизация (`/admin/login`):** Защита паролем доступа к расписанию.
2. **Панель управления (`/admin`):**
   * Просмотр списка всех услуг с детальной информацией (длительность приёма, часы начала и окончания работы).
   * **Создание услуг:** Форма с указанием названия, времени работы и длительности приёма. Сетка слотов генерируется на сервере автоматически.
   * **Редактирование услуг:** Возможность изменить параметры существующей услуги через форму.
   * **Удаление услуг:** Быстрое удаление ненужного типа записи с подтверждением.

## Быстрый старт

### Шаг 1. Установка зависимостей
Установите все зависимости в корневой директории (проект настроен с использованием npm workspaces):
```bash
npm install --legacy-peer-deps
```

### Шаг 2. Запуск эмулятора API (Prism Mock)
Для проверки работы фронтенда по контракту запустите mock-сервер Prism, который будет отдавать корректные тестовые данные на основе `openapi.yaml`:
```bash
npm run api:mock
```
Prism запустится на порту `http://localhost:4000`.

### Шаг 3. Запуск фронтенда в режиме разработки
В другом окне терминала выполните команду для запуска Vite dev-сервера:
```bash
npm run frontend:dev
```
Интерфейс откроется по адресу `http://localhost:5173/` (или другому свободному порту, указанному в консоли).

## Запуск через Docker Compose (Рекомендуемый способ)

Для максимально быстрого и удобного запуска бэкенда, фронтенда и базы данных Postgres одной командой используется Docker Compose.

### Шаг 1. Запуск всех сервисов
Находясь в корневом каталоге проекта, выполните:
```bash
docker compose up --build
```

Docker Compose автоматически:
1. Поднимет контейнер PostgreSQL (на внутреннем порту `5432`, доступном извне на хосте как `7235`).
2. Соберет и запустит контейнер бэкенда FastAPI (с автоматическим запуском миграций БД и пробросом порта **`7100`** наружу).
3. Соберет и запустит контейнер фронтенда Nginx (с пробросом порта **`80`** наружу).

### Шаг 2. Использование и проверка
* **Фронтенд (Интерфейс пользователя):** Откройте в браузере [http://localhost](http://localhost) (или [http://localhost:80](http://localhost:80)).
* **Бэкенд API:** Доступен по адресу [http://localhost:7100](http://localhost:7100).
* **Интерактивная документация API (Swagger):** Откройте в браузере [http://localhost:7100/docs](http://localhost:7100/docs).
* **Подключение к БД напрямую:** `postgresql://postgres:postgres@localhost:7235/zapisi`.

### Управление контейнерами
* Чтобы остановить контейнеры, нажмите `Ctrl+C` в терминале или выполните:
  ```bash
  docker compose down
  ```
* Чтобы остановить контейнеры со сбросом данных базы данных (очисткой тома `pgdata`):
  ```bash
  docker compose down -v
  ```

## Полезные команды

* `npm run api:compile` — Скомпилировать контракт TypeSpec (`main.tsp`) в OpenAPI.
* `npm run api:mock` — Запустить эмулятор Prism Mock API (порт 4000).
* `npm run frontend:dev` — Запустить dev-сервер фронтенда.
* `npm run frontend:build` — Собрать фронтенд для продакшна в папку `frontend/dist`.
* `npm run frontend:lint` — Запустить линтер ESLint для проверки качества кода фронтенда.
* `docker compose up --build` — Собрать и запустить весь стек приложений.
* `docker compose down` — Остановить все запущенные контейнеры.
